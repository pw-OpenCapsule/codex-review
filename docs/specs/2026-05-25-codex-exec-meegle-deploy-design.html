设计稿：Codex Exec 复核 + Meegle 自动建缺陷 + 远程部署

日期：2026-05-25 · 作者：leo + Claude · 项目：codex-review

1. 背景

当前 send_lark_report.sh 依赖外部 codex.leeguoo.com 把 codex 原始审查结果翻译成中文 P0–P5 摘要。本次改造将三件事一起做：

  1. 把翻译换成本地 codex exec 在已同步的镜像 repo 里做复核 + 翻译（剔除幻觉、保留真实问题）；
  2. 飞书消息发送成功后自动调用 meegle-cli，每个 P0–P5 问题各建一条缺陷到 Meegle「Code Review」项目；
  3. 生产运行迁移到 192.168.0.190（Mac，已装 codex / meegle CLI），本地只负责写代码并通过 GitHub 部署。

4. Codex Exec 复核 + 翻译

4.1 调用形态

cd "${WORKDIR}/${repo}"
git checkout "${PR_HEAD_SHA}"   # blame / 读源码所需

codex exec --skip-git-repo-check --output-last-message /tmp/codex-out.txt - <<EOF
${PROMPT_HEADER}
${REVIEW_TEXT}
EOF

4.2 Prompt 设计

你是代码审查复核员。下面是 codex 对 PR 的原始审查输出。
任务：
1. 在当前工作目录读取被指出的代码，验证每个问题是否真实存在
2. 剔除幻觉 / 误报（描述与代码不符、文件不存在、行号无意义等）
3. 保留下来的问题翻译成中文，按 P0–P5 分级

严格输出 JSON（不要任何额外文字）：
{
  "issues": [
    { "severity": "P0|P1|P2|P3|P4|P5",
      "summary_zh": "中文问题描述（保留反引号包裹的标识符）",
      "file": "相对路径",
      "line_start": <int>,
      "line_end": <int>,
      "evidence": "复核理由" }
  ],
  "rejected": [
    { "severity": "P?", "original": "原文摘录", "reason": "为什么判为误报" }
  ]
}

原始审查：{REVIEW_TEXT}
PR 元信息：仓库={REPO} 分支={BRANCH} PR={PR_NUMBER} HEAD={SHA}

4.3 错误处理

情况行为
codex exec 超时（默认 300s）退到「无摘要直发原文」模式，日志告警
codex 输出非 JSON同上，raw 写入 STATE_DIR/codex-bad-output/<repo>-<pr>.txt
issues 为空但 rejected 非空跳过 issue 列表，仅发 PR 链接，Meegle 也不建单
codex 二进制不存在启动时检测，缺则 fail send_lark_report 整个流程

5. Meegle 自动建缺陷

5.1 触发与幂等

  • 触发：飞书 webhook 返回成功 且 issues 非空。
  • 幂等：${STATE_DIR}/meegle-created.tsv 记录 pr_url \t issue_index \t work_item_id，重跑时命中跳过。
  • 粒度：每个 issue 一条 bug。

5.2 字段映射

Meegle field_key来源示例
name[${repo}#${PR}] ${summary_zh[:60]}[xhs-toolkit#42] login 401 后无错误提示
description多行模板（见 5.3）—
current_status_operatorblame → LARK_USER_MAP → meegle_user_key["7368413876789141507"]
severityP0–P5 → select option_id（启动时一次性查 meta-fields 建表）—
tags固定 codex-review + 仓库名["codex-review", "xhs-toolkit"]

5.3 description 模板

## 来源
- 仓库: ${repo}
- PR: ${pr_url}
- 文件: ${file}:${line_start}-${line_end}
- 引入: ${blame_author} @ ${blame_sha} (${blame_date})

## 复核结论
${codex_evidence}

## 原始审查摘录
${original_review_excerpt}

5.4 责任人三级兜底

  1. git blame ${file} -L${line_start},${line_end} → email
  2. email → LARK_USER_MAP → lark_user_id → meegle_user_key（meegle CLI 复用 Lark 身份）
  3. 任意一步失败 → MEEGLE_DEFAULT_ASSIGNEE
  4. 仍失败 → 建无 assignee 的 bug（丢单比建错单更糟，宁愿留空）

6.2 新增 scripts/deploy.sh

#!/usr/bin/env bash
set -euo pipefail
REMOTE="leo@192.168.0.190"
REMOTE_DIR="/Users/leo/codex-review"

# 1. 远程冲突检查
if ! ssh "${REMOTE}" "cd ${REMOTE_DIR} && git status --porcelain | wc -l | grep -q '^ *0$'"; then
  echo "❌ Remote has local changes, aborting"; exit 1
fi
# 2. 撞 cron 检查
if ssh "${REMOTE}" "pgrep -f 'daily_review.sh|send_lark_report.sh' >/dev/null"; then
  echo "❌ Cron job currently running on remote, retry later"; exit 1
fi
# 3. 本地 commit + push
git add -A
git commit -m "${1:-deploy: $(date +%F\ %T)}" || echo "(nothing to commit)"
git push origin main
# 4. 远程拉取
ssh "${REMOTE}" "cd ${REMOTE_DIR} && git pull --ff-only"
echo "✅ Deployed"
